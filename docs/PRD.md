# 마이링크 (MyLink) 요구사항 정의서 (PRD)
- **문서 버전:** v1.3.0

## 1. 프로젝트 개요
- **프로젝트명:** 마이링크 (MyLink)
- **목적:** 사용자가 자신의 포트폴리오, 블로그, 비즈니스 웹사이트 등을 하나의 링크로 통합하여 공유할 수 있는 프로필 링크 멀티플렉싱 서비스
- **대상 사용자:** 
  - 유튜브, 인스타그램 등 여러 채널을 운영하며 통합 링크를 공유하는 **크리에이터 및 인플루언서**
  - 자신만의 포트폴리오나 비즈니스 정보를 깔끔하게 전달하고자 하는 **비즈니스 전문가**

## 2. 필수 및 제외 기능

### 2.1 핵심 기능 (MVP)
1. **사용자 인증 (Google Auth & 닉네임 기반 URL)**
2. **관리자 대시보드 (프로필 및 링크 인라인 편집 CMS)**
3. **퍼블릭 사용자 페이지 (외부 공유용, 모바일 최적화)**

### 2.2 제외 기능 (Not in Scope)
- **프로필 이미지 직접 업로드:** 스토리지 구성의 단순화와 가벼운 구조를 위해 직접 업로드 제외. (단, **구글 계정의 프로필 이미지**는 자동으로 불러와 표출)
- **개발자 특화 카드:** 개발자 전용 특화 블록(Github 상태창 등)은 범용성을 위해 제외.
- **링크 드래그 앤 드롭 기능:** 링크 순서를 드래그로 변경하는 기능은 초기 버전의 단순한 관리를 위해 제외.
- **모바일 뷰 미리보기 (Live Preview):** 복잡성을 피하기 위해 개발 범위에서 제외.
- **링크 활성화/비활성화 토글:** 불필요한 관리를 줄이고 간소화하기 위해 제외.
- **다양한 테마 시스템 및 고급 블록(미디어 임베딩 등):** 필수 요구사항에 부합하지 않으므로 제외.
- **이메일/비밀번호 기반 회원가입:** 복잡한 가입 절차를 배제하고 구글 로그인 단일화.

## 3. 기능 및 구조 상세 설명

### 3.1 사용자 인증 (Authentication)
- **기능 설명:** Firebase Authentication을 연동한 구글 계정 간편 접속.
- **세부 요구사항:**
  - 사용자 편의성 및 보안을 고려하여 구글 계정 원클릭 로그인(Google Auth) 전용으로 구축.
  - 가입 시 접속 URL(슬러그)은 별도의 아이디가 아닌 사용자의 **닉네임(displayName)** 을 사용합니다. (결과 위치: `domain.com/displayName`).
  - 초기 `displayName` 값은 구글 로그인 시 가져온 **지메일(Gmail) 주소의 '@' 앞부분**을 추출하여 시스템이 자동으로 설정합니다.

### 3.2 관리자 대시보드 (Dashboard & CMS)
- **기능 설명:** 사용자의 정보와 링크 목록을 제어하며, **모든 수정 작업은 모달 없이 즉시 인라인 편집(Inline Edit)** 방식으로 동작합니다.
- **프로필 관리:**
  - 별도의 이미지 업로드 없이 **구글 프로필 이미지(photoURL)** 를 자동 연동하여 표시.
  - 화면에 보여지는 실제 이름(displayName) 및 짧은 자기소개(Bio)를 클릭하여 직접 인라인 편집으로 수정.
- **링크 블록 관리 (CMS):**
  - 새로운 링크 항목(제목, 목적지 URL) 추가 및 삭제 기능.
  - 등록된 기존 링크의 제목, URL 역시 클릭하여 인라인 수정 가능.
  - **파비콘 가져오기:** 사용자가 목적지 URL을 등록/저장하면 **구글 파비콘 API (Google Favicon API)** 를 통해 자동으로 아이콘을 추출하여 링크 좌측에 삽입.

### 3.3 데이터베이스 모델링 구조 (NoSQL - Firestore)
- **기능 설명:** Firebase Firestore 기반의 사용자 컬렉션 및 링크 서브컬렉션 노드 설계.
- **상세 스키마(예시):**
  ```json
  "Users Collection": {
    "{uid}": {
      "displayName": "gmail_prefix", // 이메일 앞부분으로 자동 초기화. 웹 URL 슬러그 및 화면 노출 이름으로 동시 사용됨
      "photoURL": "https://...",   // 구글 Auth 로부터 연동받은 구글 설정 이미지
      "bio": "짧은 자기소개",
      "createdAt": "timestamp"
    }
  },
  "Links Sub-collection (Users/{uid}/links)": {
    "{link_id}": {
      "title": "링크 제목",
      "url": "https://example.com",
      "faviconUrl": "https://s2.googleusercontent.com/s2/favicons?domain=example.com",
      "createdAt": "timestamp",
      "clickCount": 0  // 나중에 조회수 통계를 더하기 위해 기본값 0으로 세팅
    }
  }
  ```

### 3.4 디자인 시스템 (Design System)
- **기능 설명:** shadcn/ui 기반의 퍼블릭 뷰 및 대시보드 환경 컴포넌트 프레임워크.
- **세부 요구사항:**
  - **디자인 톤앤매너:** 군더더기 없이 심플하고, 신뢰감을 주는 '전문적인 기업 사이트 느낌'의 정돈된 UI 지향.
  - **반응형 및 모바일 최적화:** 
    - 최종 공유되는 '퍼블릭 사용자 페이지'는 철저하게 모바일 환경을 기준으로 최적화시킴.
    - PC 브라우저로 접속할 경우 화면 가득 차는 형태가 아닌, 모바일 규격 고정 폭(가운데 정렬)으로 표시하여 깔끔한 비율을 유지.
