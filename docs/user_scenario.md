# 마이링크 (MyLink) 사용자 시나리오 (User Scenario)

- **작성일:** 2026-04-05
- **관련 PRD:** v1.3.0 기준
- **시나리오 작성 형식:** "사용자는 [목적/의도]하기 위해 [행동/액션]한다."

---

## 1. 방문자 (Visitor) 시나리오

### 1-1. 정상적인 페이지 접속 및 링크 확인
- **사용자는** 크리에이터의 여러 외부 채널 및 비즈니스 정보를 한눈에 확인하기 위해 **제공된 마이링크 고유 URL(`domain.com/displayName`)을 클릭하여 사이트에 접속한다.**
- **사용자는** 접속하자마자 기다릴 필요 없이 즉시 내용을 확인하기 위해 **심플하게 텍스트와 파비콘 위주로 렌더링된 인덱스 화면을 스크롤한다.**
- **사용자는** 접속한 링크의 주인이 맞는지 식별하기 위해 **페이지 상단에 노출된 '구글 프로필 이미지'와 '이름(displayName)', '소개말(Bio)'을 읽는다.**

### 1-2. 링크 이동
- **사용자는** 자신이 원하는 특정 서비스(웹사이트, 포트폴리오 등)로 이동하기 위해 **원하는 버튼에 부착된 '파비콘(Favicon)' 및 '링크 제목'을 확인하고 버튼을 탭(클릭)한다.**
- **사용자는** 링크를 누른 후 원래 마이링크 페이지를 놓치지 않기 위해, **기존 창이 유지된 상태로 별도의 새 창(새 탭, Target Blank 속성)이 열려 해당 목적지 사이트가 뜨는 것을 확인한다.**

### 1-3. 오류 페이지(404) 접속 대응
- **사용자는** 실수로 오타를 내거나 더이상 존재하지 않는 사용자의 URL(`domain.com/없는이름`)로 접속했을 때, **현 페이지가 유효하지 않음을 인지하기 위해 "존재하지 않는 페이지입니다" 안내 문구가 포함된 404 에러 화면을 마주한다.**

### 1-4. 무단 클릭(어뷰징) 시도 대응
- **사용자는** 악의적인 목적이나 실수로 동일한 버튼을 수십 번 연타하더라도, **시스템이 단기 연속 중복 클릭을 무시해 실제 링크 `clickCount` 통계가 무의미하게 부풀려지지 않도록 조치되는 과정을 인지하지 못한 채 자연스럽게 사이트를 이용한다.**

## 2. 시스템 (System) 자동화 시나리오

### 2-1. 클릭수 통계 자동 카운트
- **시스템은** 나중에 소유자에게 유의미한 클릭 데이터를 제공하기 위해, **방문자가 외부 링크를 클릭하여 넘어가는 순간 데이터베이스 해당 링크 문서의 `clickCount`를 1 증가시키는 이벤트를 백그라운드에서 조용히 실행한다.** (방문자 전체 카운트는 제외)

### 2-2. 파비콘(Favicon) 자동 생성
- **시스템은** 소유자가 링크 목적지 URL 입력을 완료하고 저장하는 순간, **해당 주소의 도메인을 구글 파비콘 API에 던져 아이콘 이미지를 자동 추출하고 UI 상 버튼 좌측에 깔끔하게 렌더링한다.**

### 2-3. 목적지 URL 자동 보정 (http/https)
- **시스템은** 소유자가 URL 입력 시 실수로 `naver.com`과 같이 프로토콜을 생략하고 작성했을 때, **링크 오류를 막기 위해 찰나의 순간에 스스로 주소 앞에 `https://`를 강제로 덧붙여(Prepend) 안전하게 DB에 저장한다.**

## 3. 소유자 (Owner) 관리 시나리오

### 3-1. 최초 가입 세팅 및 로깅
- **사용자는** 자신만의 관리자 대시보드에 안전하게 접근하기 위해 **"Google 계정으로 시작하기" 버튼을 클릭해 소셜 연동(OAuth)을 승인한다.**
- **사용자는** 최초 가입 시 본인만의 고유 URL 슬러그를 획득하기 위해 **시스템이 지메일(Gmail) 아이디 앞부분을 자동 파싱하여 설정한 `displayName`을 초기값으로 부여받고 텅 빈 대시보드 화면에 진입한다.**

### 3-2. 일반 로그인 및 내 페이지 점검
- **사용자는** 며칠 뒤 자신의 링크를 더 추가하기 위해 **다시 구글 로그인을 눌러 비밀번호 입력 통과 없이 대시보드에 즉시 진입한다.**
- **사용자는** 텍스트를 수정하거나 링크를 추가한 뒤 방문자들이 내 페이지를 어떻게 보는지 점검하기 위해 **대시보드 상단 구석의 '내 페이지 보기' 아이콘을 클릭하여 자신의 퍼블릭 페이지 URL 탭을 띄운다.**

### 3-3. 대시보드 프로필 수정 (인라인 편집)
- **사용자는** 구글 지메일 아이디로 만들어진 현재 URL/이름을 다른 닉네임으로 바꾸기 위해 **대시보드의 '이름(displayName)' 텍스트를 마우스로 곧바로 클릭하여(인라인 편집 활성화) 새로운 영문/한글 이름으로 타이핑한다.**
- **사용자는** 방문자에게 목적을 어필하기 위해 **빈 칸으로 남겨진 '자기소개(Bio)' 영역을 클릭해 안내 문구를 입력한다.**
- **사용자는** 수정한 텍스트 데이터를 저장하기 위해 **엔터(Enter) 키를 누르거나 화면 빈 곳을 포커스 아웃(클릭)하고, 저장이 완료됨을 알리는 "저장되었습니다" 알림(Toast) 또는 체크 아이콘(✓)이 1초간 깜빡이는 것을 눈으로 확인한다.**

### 3-4. 새로운 링크 블록 추가 (CMS)
- **사용자는** 방문자에게 새 외부 웹사이트를 제공하기 위해 **화면 하단의 '새 링크 추가' 버튼을 눌러 빈 텍스트 상태의 링크 블록 행(Row)을 1개 생성한다.**
- **사용자는** 해당 링크가 어떤 곳인지 명시하기 위해 **'제목 없음' 텍스트 영역을 인라인 클릭하여 '내 최신 유튜브 영상' 같이 목적지 제목을 입력하고 엔터를 친다.**
- **사용자는** 해당 버튼이 실제로 이동할 주소를 세팅하기 위해 **'URL 입력' 영역을 클릭하여 웹 주소를 붙여넣기(Paste)하고 화면 바깥을 클릭해 성공 알림을 받는다.** (이후 2-2 시스템 자동화를 통해 파비콘이 뜹니다.)

### 3-5. 기존 링크 수정 (인라인 편집)
- **사용자는** 이전에 올려둔 링크 제목에 오타가 있음을 발견하고 इसे 고치기 위해 **해당 링크 카드의 제목 텍스트 위치를 바로 클릭하여 새로운 내용으로 덮어쓰고 엔터를 누른다.**
- **사용자는** 내용이 서버에 제대로 반영되었는지 안심하기 위해 **수정 및 엔터를 치는 순간 노출되는 시각적 피드백(저장 완료 토스트 배너)을 확인하고 다음 작업으로 넘어간다.**

### 3-6. 기존 링크 블록 삭제
- **사용자는** 이벤트가 만료되었거나 삭제하고 싶은 링크를 마이링크 메인 화면에서 지우기 위해 **해당 링크 블록 우측 끝 영역에 위치한 '휴지통(삭제)' 아이콘을 누른다.**
- **사용자는** 실수로 중요한 정보가 지워지는 대참사를 막기 위해 **화면 내에 팝업된 "해당 링크를 정말 영구 삭제하시겠습니까?" 모달 경고창에서 '확인(Yes)' 버튼을 명확히 누른 뒤 카드가 소멸하는 것을 확인한다.**

### 3-7. 접속 문자(닉네임) 중복 생성 대응
- **사용자는** 첫 구글 가입 시 내 이메일 앞자리(예: `john`)를 누군가 이미 사용중일 경우, **시스템이 URL 충돌을 방지하기 위해 내 접속 주소를 `john_1` 등 난수 조합으로 틀어 생성한 것을 확인하고 대시보드에서 내가 원하는 유니크한 이름으로 재수정한다.**

### 3-8. 빈칸(Empty Value) 저장 오류 방어
- **사용자는** 인라인 편집 시 실수로 목적지 URL이나 제목을 다 지우고 빈 칸으로 둔 채 밖을 클릭(저장)할 때, **시스템이 "내용을 입력해주세요" 경고 알림(Toast)을 띄움과 동시에 저장을 거부하여 사이트 레이아웃의 투명 붕괴 현상을 사전에 방어해주는 것을 경험한다.**
